# 启动说明

## 修复内容

已在原有的 `frontend/app.py` 中添加CORS支持,无需创建新文件。

## 启动步骤

### 1. 安装后端依赖 (已完成✅)

所有依赖已通过 `uv` 安装完成:
- flask
- flask-cors
- pandas
- openpyxl
- requests
- scikit-learn
- matplotlib
- seaborn

### 2. 启动后端服务

```bash
cd /Users/jove/Lab/Jove-Programming-Diary/webpack/customer-complaint-check-main/frontend
uv run python app.py
```

后端服务将在 `http://localhost:5001` 启动。

**注意**: 必须使用 `uv run python` 而不是直接使用 `python`,因为依赖安装在uv管理的环境中。

### 3. 启动前端服务

新开一个终端窗口:

```bash
cd /Users/jove/Lab/Jove-Programming-Diary/webpack/customer-complaint-check-main/manager-web
npm run dev
```

前端服务将在 `http://localhost:5173` 启动。

### 4. 访问应用

浏览器打开: `http://localhost:5173/`

## 配置说明

### 后端配置 (frontend/app.py)
- 端口: 5001
- CORS允许的源: `http://localhost:5173`, `http://127.0.0.1:5173`

### 前端配置 (manager-web/src/config/env.ts)
- API地址: `http://localhost:5001`

## 验证

启动后检查:
1. 后端终端显示服务运行在 port 5001
2. 浏览器控制台无跨域错误
3. Network面板显示API请求成功(状态码200)
